# [팀원 공유용] OmniSite 파트별 세부 업무분장 및 개발 가이드 가이드라인 (v1.0.0)

본 가이드는 다목적 스마트시티 공간 의사결정 플랫폼 **'OmniSite'**의 성공적인 완수를 위해 **조장을 제외한 7인의 개발팀원(DH, KM, 학부생 B, CM, SH, Cj, MY)**이 각자 지망한 파트에서 완수해야 할 기술 스펙과 개발 흐름을 정의한 가이드라인입니다.

팀원들은 본 가이드를 정독한 후, 자신의 파트에서 정의된 **[Input ➔ Action ➔ Output]** 흐름에 맞춰 독립 개발 및 단위 테스트를 수행해 주시기 바랍니다.

---

## 📂 [AI 파트] 1. AI (RAG, LangGraph) — 담당: 주니어 A (DH)

### 📌 개요 및 수행 역할
*   **목표:** RAG 기반 스마트 쉼터/흡연부스 설치 조례 데이터를 기반으로 찬성/반대 주민 및 공무원의 3자 모의 토론 시뮬레이션(LangGraph)을 구축하고, 이를 실시간 스트리밍(SSE)으로 프론트엔드에 송출합니다.

### 📥 1. 입력 데이터 및 환경 (Inputs)
*   **조례 데이터셋:** MY(프롬프트)가 청킹 및 마크다운 정제를 완료한 조례집 텍스트.
*   **컨텍스트 벡터:** SH(AHP/인자)가 산출해 준 특정 필지의 지역 갈등 민감도 벡터 점수 ($CSS$ 등급: 상/중/하).
*   **환경 설정:** OpenAI API Key (`GPT-4o` 모델 고정 사용).

### ⚙️ 2. 상세 구현 방법 및 기술 스펙 (Actions)
1.  **LangGraph State 워크플로우 정의:**
    *   토론 참가자의 대화 상태(`State`)를 관리할 `TypedDict` 클래스를 정의합니다.
    *   `citizen_agent`(주민대표 - 반대), `merchant_agent`(상인대표 - 찬성), `official_agent`(조정 공무원) 노드를 생성합니다.
    *   주입받은 갈등 민감도($CSS$)가 "상"일 경우, `citizen_agent` 프롬프트의 분노/반대 온도를 극한으로 끌어올리고, "하"일 경우 합의 의지를 높이도록 동적 프롬프트 인젝션을 수행합니다.
2.  **FastAPI SSE (Server-Sent Events) 스트리밍 구현:**
    *   LangGraph의 각 노드 실행 결과가 떨어질 때마다 `async for` 문을 이용해 스트리밍 제너레이터를 가동합니다.
    *   FastAPI의 `sse_starlette` 패키지 또는 `StreamingResponse`를 사용하여 `text/event-stream` 규격으로 클라이언트에 실시간 바이트 전송을 수행합니다.
    *   데이터 전송 규격: `data: {"sender": "주민대표 (반대)", "text": "학교 정화구역 인근에..."}`

### 📤 3. 최종 산출물 (Outputs)
*   **소스 파일:** `/app/api/simulation.py` (FastAPI SSE 라우터), `/app/ai/langgraph_chain.py` (토론 그래프 엔진).
*   **API 엔드포인트:** `GET /api/v1/simulation/stream?parcel_id={parcel_id}` (실시간 토론 스트림 데이터).

## 🗄️ [DB 파트] 2. DB 및 공간 연산 — 담당: 학부생 KM (메인) & 학부생 B

### 📌 개요 및 수행 역할
*   **목표:** 데이터 정제가 끝난 용산구 지적도 및 시설물 정보를 PostGIS 공간 데이터베이스에 적재하고, 버스정류장/지하철/스쿨존의 법정 금역 범위를 계산하여 제외(Exclusion) 필지 목록을 연산하는 쿼리를 작성합니다.

### 📥 1. 입력 데이터 및 환경 (Inputs)
*   **공간 원본:** 정제팀이 정제한 지하철역 출입구, 초등학교/유치원 경계, 용산구 연속지적도 파일 (SHP 및 CSV 포맷).
*   **환경 설정:** PostgreSQL 15+ & PostGIS 확장 모듈이 켜진 로컬 DB 서버.

### ⚙️ 2. 상세 구현 방법 및 기술 스펙 (Actions)
1.  **PostGIS 기반 DDL 및 데이터 적재:**
    *   지적 폴리곤 적재를 위한 `cadastral_lands` 테이블 스키마를 정의합니다. 반드시 공간 칼럼 `geom GEOMETRY(MultiPolygon, 4326)`를 포함해야 합니다.
    *   버스정류장/지하철 출입구 포인트(`geom GEOMETRY(Point, 4326)`) 테이블을 개설하고 데이터를 임포트합니다.
2.  **공간 차집합 배제 연산 구현:**
    *   지하철역 출입구로부터 10m 버퍼(`ST_Buffer(geom, 0.00009)`) 및 스쿨존 경계선으로부터 200m 버퍼(`ST_Buffer(geom, 0.0018)`)를 계산합니다.
    *   지하철/스쿨존 금역 레이어들을 `ST_Union`으로 합친 뒤, 전체 지적도 테이블과 `ST_Difference` 연산을 돌려 금지 구역에 걸쳐있거나 포함되는 필지를 완전 탈락시킵니다.
3.  **렌더링 병목 해소 (KM 전담):**
    *   프론트엔드 Mapbox/Leaflet에 전달할 GeoJSON 크기를 대폭 줄이기 위해 `ST_Simplify(geom, 0.0001)` 처리를 통해 geometry 정점 개수를 최적화합니다.
    *   연산 속도 단축을 위해 `CREATE INDEX idx_cadastral_geom ON cadastral_lands USING GIST(geom);` 공간 인덱싱을 강제 적용합니다.

### 📤 3. 최종 산출물 (Outputs)
*   **소스 파일:** `/db/migrations/schema.sql` (물리 테이블 스키마), `/db/queries/exclusion_logic.sql` (배제 필지 선별 SQL).
*   **최종 결과:** 공간 차집합 연산 수행 후 도출된 용산구 적격 필지 테이블 데이터셋.

## 💻 [프론트/풀스택 파트] 3. FullStack (Next.js - API & 맵 연동) — 담당: 주니어 B (CM)

### 📌 개요 및 수행 역할
*   **목표:** 제공받은 Next.js 프로토타입 코드를 백엔드 API 규격서에 맞춰 동적 데이터 바인딩을 완료하고, 마커 드래그 보정(HITL) 및 AI 토론 실시간 SSE 타이핑 연출을 제어합니다.

### 📥 1. 입력 데이터 및 환경 (Inputs)
*   **프론트 템플릿:** 1회차 개발 단계에서 제작된 Next.js 기반 UI 프로토타입 소스코드.
*   **연동 대상:** Cj(FastAPI)와 SH(AI/AHP)가 구축한 백엔드 REST API 인터페이스.

### ⚙️ 2. 상세 구현 방법 및 기술 스펙 (Actions)
1.  **Axios API 연동 및 상태 전이:**
    *   Step 1 파일 업로드 드롭존 이벤트를 백엔드 파일 업로드 API(`POST /api/v1/upload`)로 바인딩합니다.
    *   Step 3 AHP 쌍대비교 입력 상태(`ahpWeights`)가 변경될 때마다 백엔드로 Axios 요청을 쏘아 리턴되는 일관성 비율(C.R.)이 `0.1` 이상이면 에러 메시지를 노출하고 저장 버튼을 비활성화 처리합니다.
2.  **Leaflet 맵 렌더링 동기화 및 HITL 보정:**
    *   Step 2 화면에서 마커 핀을 마우스 드래그앤드롭하여 옮길 때, 드래그가 끝난 시점(`dragend`)의 위경도 좌표를 백엔드 보정 API(`POST /api/v1/hitl/commit`)로 전송하여 DB의 임시 좌표를 수정합니다.
3.  **SSE 실시간 타이핑 렌더러 탑재:**
    *   Step 5 시뮬레이션 진입 시 브라우저 내장 `EventSource` 객체를 인스턴스화하여 DH(RAG/LangGraph)가 쏘는 SSE 스트림 엔드포인트를 구독합니다.
    *   스트림 메시지가 들어올 때마다 타이핑 사운드 효과(옵션) 및 텍스트 슬라이스 처리를 가동하여 채팅 창에 실시간 중계 화면을 구축합니다.

### 📤 3. 최종 산출물 (Outputs)
*   **소스 파일:** `/frontend/src/app/page.js` 및 컴포넌트 폴더 내 React 파일 일체.
*   **최종 결과:** 백엔드와 연동되어 데이터 업로드부터 최종 PDF 보고서 출력까지 전 주기가 실시간 구동되는 완성형 대시보드 웹 어플리케이션.

## ⚙️ [AI/수리 파트] 4. AI (감리, 페르소나, 검증, AHP) — 담당: 주니어 C (SH)

### 📌 개요 및 수행 역할
*   **목표:** 파일 업로드 시 AI가 내용을 식별하는 지능형 감리 파이프라인을 구축하고, AHP 행렬 연산과 C.R. 계산 모듈을 파이썬으로 구현하며, 사후 완료 공문서를 판독하는 Audit AI를 개발합니다.

### 📥 1. 입력 데이터 및 환경 (Inputs)
*   **AHP 입력:** 프론트엔드에서 날아온 5대 평가 인자 쌍대비교 가중치 객체.
*   **감리 대상:** 지목 주소 등이 담긴 업로드 파일 5행 스니펫 텍스트.
*   **공문서:** 업로드된 준공계/보고서 실물 PDF 문서.

### ⚙️ 2. 상세 구현 방법 및 기술 스펙 (Actions)
1.  **AHP 엔진 및 일관성 검증 모듈 개발:**
    *   가중치 척도 값을 토대로 일관성 지수($CI = \frac{\lambda_{max} - n}{n - 1}$)와 무작위 지수($RI$)를 이용해 일관성 비율($CR$)을 산출하는 파이썬 코드를 빌드합니다.
    *   `C.R. >= 0.1` 일 경우 즉시 예외 오류(`HTTP 422`)를 반환하는 디펜시브 코드를 심고, 검증 통과 시 `ahp_models` 테이블에 `is_locked = TRUE` 상태로 영구 보관합니다.
2.  **AI 시맨틱 감리 및 Audit AI 모델 설계:**
    *   업로드된 CSV 내용이 스마트 부스 필수 지표(유동인구, CCTV 위치 등)에 부합하는지 템플릿 일치 여부를 AI 감리 엔진이 판정하도록 프롬프트를 튜닝합니다.
    *   준공 PDF 보고서 업로드 시 OCR 팩트체크 엔진을 가동하여 실제 부스 위치 위경도 텍스트를 추출하고, 시나리오 매핑 코드를 설계합니다.

### 📤 3. 최종 산출물 (Outputs)
*   **소스 파일:** `/app/services/ahp.py` (AHP 수리 연산 모듈), `/app/services/audit.py` (AI 감리 및 OCR 판독 모듈).
*   **최종 결과:** AHP 검증 통과 시 DB 락 적재 및 결측치가 없는 데이터 감리 결과 리턴 모듈.

## 🌐 [백엔드 라우터 파트] 5. FastAPI 라우터 및 공통 API — 담당: 비전공 A (Cj)

### 📌 개요 및 수행 역할
*   **목표:** 회원가입/로그인 인증 파이프라인을 잡고, 파일 업로드 시 확장자 분석 분기 처리 및 카카오맵/Vworld 지오코딩 API 외부 연동 라우터를 구현합니다.

### 📥 1. 입력 데이터 및 환경 (Inputs)
*   **사용자 입력:** 회원가입 이메일, 패스워드 원본, 업로드 파일 파일 스트림.
*   **외부 API Key:** 카카오 로컬 REST API Key 또는 국토교통부 Vworld API Key.

### ⚙️ 2. 상세 구현 방법 및 기술 스펙 (Actions)
1.  **인증 및 비밀번호 단방향 암호화:**
    *   `passlib` 라이브러리의 `bcrypt` 컨텍스트를 활용하여 가입 시 비밀번호를 안전하게 단방향 암호화(해싱)한 뒤 DB에 적재합니다.
    *   `PyJWT`를 가동하여 로그인 검증 시 `Access Token`을 발행하고, API 요청 헤더의 Bearer 토큰 유효성을 판독하는 데코레이터 함수를 생성합니다.
2.  **확장자 기반 파일 분기 라우터:**
    *   FastAPI의 `UploadFile`로 수신된 파일의 확장자를 체크합니다.
    *   `.pdf` 및 `.hwp`는 AI RAG 파트(DH)로 라우팅하고, `.csv` 및 `.shp`는 DB 파트(KM)로 연동해 테이블 파싱을 구동시킵니다.
3.  **외부 지오코딩 Open API 연동:**
    *   CSV 파일 파싱 중 위경도가 공백이고 지번 주소(`jibun`)만 발견될 시, Kakao Local API (`https://dapi.kakao.com/v2/local/search/address.json`)를 비동기 호출(`httpx` 사용)하여 위경도 좌표로 즉각 치환합니다.

### 📤 3. 최종 산출물 (Outputs)
*   **소스 파일:** `/app/api/auth.py` (회원/로그인), `/app/api/upload.py` (파일 라우터), `/app/utils/geocoder.py` (외부 지오코더).
*   **최종 결과:** 회원 보안 인증 처리 완결 및 업로드된 주소 텍스트의 위경도 데이터화 성공.

## 📝 [AI 보조 파트] 6. 프롬프트 및 AI 보조 — 담당: 비전공 B (MY)

### 📌 개요 및 수행 역할
*   **목표:** AI 토론에 주입될 조례 텍스트 문서를 의미 단위로 정밀하게 쪼개는 RAG 데이터 전처리를 담당하고, 에이전트 페르소나 시스템 프롬프트 작성과 수치 정합성 테스트를 지원합니다.

### 📥 1. 입력 데이터 및 환경 (Inputs)
*   **법률 원본:** 국회 및 지자체 스마트시티, 도로점용, 금역 구역에 관한 조례집 PDF/HWP 원본 문서.
*   **테스트 케이스:** AHP 가중치 수학 연산 결과 예측 엑셀 시트.

### ⚙️ 2. 상세 구현 방법 및 기술 스펙 (Actions)
1.  **RAG용 조례 텍스트 청킹(Chunking) 및 구조화:**
    *   조례집 법 조항 텍스트를 AI 임베딩이 정확하게 인덱싱할 수 있도록 마크다운(`## 제1조`) 구조로 분리하고 불필요한 줄바꿈이나 노이즈 텍스트를 완전히 정제합니다.
2.  **AI 페르소나 시스템 프롬프트 작성:**
    *   DH(LangGraph)가 사용할 주민대표, 상인대표, 조정관의 시스템 프롬프트 가이드라인 텍스트 파일을 구체적인 성격 조건(말투, 인용할 법적 근거, 타결 타협도 점수)을 담아 작성합니다.
3.  **검증용 QA 테스트 케이스 설계:**
    *   AHP 일관성 비율 계산 모듈(SH 개발분)의 정상 작동 여부를 사전에 검증할 수 있도록 정상 수치 셋트와 에러가 나야 하는 비일관적 수치 셋트(예: C.R. >= 0.1)를 미리 엑셀 시나리오로 만들어 개발진에게 제공합니다.
    *   개발 완료 단계에서 위경도가 빈 군부대 좌표 유입 시 앱 크래시가 나지 않는지 예외 케이스 테스트 검수를 전담 수행합니다.
