# 📋 E2E 통합 실증 및 Walkthrough (v3.8.0)

스마트시티 다목적 의사결정시스템(SDSS)의 **"실제 지리적 법정동 경계 적용을 통한 주소-지도 매핑 불일치 해소"**, **"다회차(Multi-turn) AI 모의 토론 심의 시나리오 고도화"**, **"프론트엔드 Leaflet 마커 갱신 데드락 해소"** 및 **"가상 도로명주소 결함 해소 및 모의토론 완료/보고서 다운로드 활성화"** 모듈이 최종 구현되어 E2E 실증 테스트에 성공하였습니다. 

또한, v3.8.0 버전에 맞춰 사용자의 2차 피드백 사항인 **"Step 4/5 금지구역 연한 빨간색 가시화"**, **"갈등 감도 3단계 페르소나 모드 연동"**, **"RAG 오염 방지를 위한 로컬 JSON 문서 파일 격리 적재"** 사양이 완벽하게 구현 및 실증되었습니다.

---

## 🛠️ 변경 내용 요약

### 1. Step 4/5 금지구역(규제 버퍼) 옅은 빨간색 영역 가시화
- **문제점**: Step 4/5 추천 결과가 도출된 후, 추천된 3개 부지가 법적 제한 구역(금연구역, 어린이보호구역 등)을 성공적으로 피해서 선정되었음을 시각적으로 확인하기 어려웠습니다.
- **해결책**:
  - `page.js` 내부의 맵 렌더링 `useEffect`에서 `pipelineStep >= 4`일 때도 `restrictionPoints`를 순회하며 규제 시설물의 영향 반경 오버레이(`L.circle`)를 지도 배경에 추가 렌더링하도록 개선했습니다.
  - 추천 후보지의 가독성과 클릭 이벤트를 저해하지 않도록, 스타일 속성을 극도로 연한 옅은 빨간색(`color: '#f87171'`, `fillColor: '#f87171'`, `fillOpacity: 0.04`, `weight: 1`)으로 보정하여 시각화했습니다.

### 2. 모의 토론 갈등 강도 3단계(보통, 위험, 매우 위험) 페르소나 모드 적용
- **문제점**: 주민 모의 심의 토론이 지나치게 온건하고 상투적인 역할극 수준에 머물러, 실제 민감 지역의 님비(NIMBY) 갈등이나 물리적 대립 상황을 시뮬레이션하기 부적절했습니다.
- **해결책**:
  - **프론트엔드 UI**: Step 5 영역에 갈등 수준을 직관적으로 선택할 수 있는 3단계 선택기(보통 🟢, 위험 🟡, 매우 위험 🔴) 라디오 버튼 그룹을 추가하고, 선택된 강도(`intensityLevel`)를 `/spatial/debate` POST API 호출 시 payload 인자로 실어 보내도록 구현했습니다.
  - **백엔드 LLM Prompt 고도화**:
    - **보통 (Normal)**: 상생 지향, 이성적이고 정중한 말투, 신속한 조건부 합의 도출.
    - **위험 (Dangerous)**: 집값 하락 우려, 교육 정주 여건 침해 등 본격적인 님비 정서 표출, 감정적 반어법 및 격렬한 반대, 정화 기금 분담 및 이격 후퇴를 담은 까다로운 조건부 중재안 유도.
    - **매우 위험 (Very Dangerous / 교착)**: 고성과 협박조 어휘, 공권력 투입 및 강제 철거 요구, 손해배상 청구 소송 위협, 대규모 연대 규탄 집회 경고 등 파국 직전의 교착상태(Deadlock) 묘사. 주민대표단에 상시 가동 중단 폐쇄권 양임 및 헤파필터 장착 등 파격적인 타협안 체결 시나리오 설계.
  - **Mock Fallback 업그레이드**: OpenAI Key 미등록 상태로 기동하는 로컬 `mock_event_generator` 내부에 선택된 3가지 갈등 강도에 부합하는 정교한 8~9턴의 개별 시나리오를 설계하고, 실제 후보지의 공간 지표 수치(유동인구, 무단투기, 공공민원)를 실시간 바인딩하여 12글자 Yield 청크로 매끄럽게 스트리밍되도록 연동했습니다.

### 3. RAG/AI 오염 방지를 위한 로컬 JSON 문서 파일 격리 적재
- **문제점**: 모의 토론 결과를 PostgreSQL 데이터베이스 테이블에 저장할 경우, 후속 법규/조례 RAG 검색 쿼리가 가상의 대사 로그를 실존하는 법령/조례로 오인 인용(오염)할 치명적인 부작용이 발견되었습니다.
- **해결책**:
  - 데이터베이스 직접 적재 방식을 배제하여 RAG 시스템과 물리적으로 격리했습니다.
  - 토론 스트리밍이 완료되는 시점에 백엔드 로컬 파일 시스템 `backend/data/debates/` 폴더 하위에 **`debate_{pnu}_{intensity_level}.json`** 형태로 물리 문서 캐시 파일로 격리 저장하도록 파이프라인을 구축했습니다.
  - 적재 파일 내에는 후보지 정보, AHP 가중치, 선택된 갈등 강도 및 구조화된 토론 로그(`debate_logs` 배열)를 함께 저장하여 향후 게시판이나 대시보드 리더에서 독립적으로 로드해 쓸 수 있도록 조치했습니다.
