# 📋 E2E 통합 실증 및 Walkthrough (v3.9.2)

스마트시티 다목적 의사결정시스템(SDSS)의 **"실제 지리적 법정동 경계 적용을 통한 주소-지도 매핑 불일치 해소"**, **"다회차(Multi-turn) AI 모의 토론 심의 시나리오 고도화"**, **"프론트엔드 Leaflet 마커 갱신 데드락 해소"** 및 **"가상 도로명주소 결함 해소 및 모의토론 완료/보고서 다운로드 활성화"** 모듈이 최종 구현되어 E2E 실증 테스트에 성공하였습니다. 

또한, v3.8.0 및 v3.9.2 버전에 걸쳐 사용자의 피드백 사항인 **"Step 4/5 금지구역 연한 빨간색 가시화"**, **"갈등 감도 3단계 페르소나 모드 연동"**, **"RAG 오염 방지를 위한 로컬 JSON 문서 파일 격리 적재"** 사양 및 **"pgvector 타입 매칭 실패로 인한 업로드 500 에러 및 Next.js 프록시 socket hang up 장해"**, **"Custom Fetch 무한 루프로 인한 브라우저 업로드 락인 현상"**, 그리고 **"공간 입지 추천 시 금지구역 내부 필지 추천 오류 및 주변 환경 요소 편차 무시 현상"** 핫픽스가 완벽하게 해결 및 실증되었습니다.

---

## 🛠️ 변경 내용 요약

### 1. Step 4/5 금지구역(규제 버퍼) 옅은 빨간색 영역 가시화 (v3.8.0)
- **해결책**:
  - `page.js` 내부의 맵 렌더링 `useEffect`에서 `pipelineStep >= 4`일 때도 `restrictionPoints`를 순회하며 규제 시설물의 영향 반경 오버레이(`L.circle`)를 지도 배경에 추가 렌더링하도록 개선했습니다.
  - 스타일 속성을 극도로 연한 옅은 빨간색(`color: '#f87171'`, `fillColor: '#f87171'`, `fillOpacity: 0.04`, `weight: 1`)으로 보정하여 시각화했습니다.

### 2. 모의 토론 갈등 강도 3단계(보통, 위험, 매우 위험) 페르소나 모드 적용 (v3.8.0)
- **해결책**:
  - **프론트엔드 UI**: Step 5 영역에 갈등 수준을 직관적으로 선택할 수 있는 3단계 선택기(보통 🟢, 위험 🟡, 매우 위험 🔴) 라디오 버튼 그룹을 추가하고, 선택된 강도(`intensityLevel`)를 `/spatial/debate` POST API 호출 시 payload 인자로 실어 보내도록 구현했습니다.
  - **백엔드 LLM Prompt 고도화**:
    - **보통 (Normal)**: 상생 지향, 이성적이고 정중한 말투, 신속한 조건부 합의 도출.
    - **위험 (Dangerous)**: 집값 하락 우려, 교육 정주 여건 침해 등 본격적인 님비 정서 표출, 감정적 반어법 및 격렬한 반대, 정화 기금 분담 및 이격 후퇴를 담은 까다로운 조건부 중재안 유도.
    - **매우 위험 (Very Dangerous / 교착)**: 고성과 협박조 어휘, 공권력 투입 및 강제 철거 요구, 손해배상 청구 소송 위협, 대규모 연대 규탄 집회 경고 등 파국 직전의 교착상태(Deadlock) 묘사. 주민대표단에 상시 가동 중단 폐쇄권 양임 및 헤파필터 장착 등 파격적인 타협안 체결 시나리오 설계.
  - **Mock Fallback 업그레이드**: OpenAI Key 미등록 상태로 기동하는 로컬 `mock_event_generator` 내부에 선택된 3가지 갈등 강도에 부합하는 정교한 8~9턴의 개별 시나리오를 설계하고, 실제 후보지의 공간 지표 수치(유동인구, 무단투기, 공공민원)를 실시간 바인딩하여 12글자 Yield 청크로 매끄럽게 스트리밍되도록 연동했습니다.

### 3. RAG/AI 오염 방지를 위한 로컬 JSON 문서 파일 격리 적재 (v3.8.0)
- **해결책**:
  - 데이터베이스 직접 적재 방식을 배제하여 RAG 시스템과 물리적으로 격리했습니다.
  - 토론이 완료되는 시점에 백엔드 로컬 파일 시스템 `backend/data/debates/` 폴더 하위에 **`debate_{pnu}_{intensity_level}.json`** 형태로 물리 문서 캐시 파일로 격리 저장하도록 파이프라인을 구축했습니다.
  - 적재 파일 내에는 후보지 정보, AHP 가중치, 선택된 갈등 강도 및 구조화된 토론 로그(`debate_logs` 배열)를 함께 저장하여 향후 게시판이나 대시보드 리더에서 독립적으로 로드해 쓸 수 있도록 조치했습니다.

### 4. pgvector RAG/태그 연산 Type Mismatch 500 크래시 핫픽스 (v3.9.0)
- **해결책**:
  - `backend/app/routers/upload.py`와 `spatial.py` 내의 모든 `<=>` 연산 부분에 파라미터 캐스팅 `::vector` 구문을 적용했습니다 (예: `:tag_embedding::vector`, `:query_embedding::vector`). 이로써 데이터베이스 레벨에서 연산자 타입 매칭이 항상 성공하도록 안전장치를 마련했습니다.

### 5. Next.js Dev Server 프록시 루프백 ECONNRESET 해소 (v3.9.0)
- **해결책**:
  - `page.js` 파일의 모듈 레벨에 relative `/api/v1` 경로 호출 시에만 `http://127.0.0.1:8000` 백엔드로 직접 CORS 호출을 연결해 주는 **커스텀 fetch 래퍼 함수(Custom Fetch Interceptor)**를 주입했습니다.

### 6. Custom Fetch 전역 식별자 무한 루프 해소 (v3.9.1)
- **해결책**:
  - 기존 `const fetch` 명칭의 섀도잉 재정의 방식에서 독자적인 `const apiFetch` 식별자로 명칭을 분리하여 선언했습니다.
  - `page.js` 내부의 모든 비동기 백엔드 API 통신 지점(15개소)의 `fetch` 호출을 `apiFetch`로 전면 대체하여 순환 참조 무한 루프를 완벽히 해결했습니다.

### 7. 공간 입지 추천 금지구역 매핑 오류 및 후보 스캔 범위 고도화 (v3.9.2)
- **문제점**:
  1. 공간 추천 시 금지구역 배제 연산 과정에서 데이터가 없는 비어 있는 `childcare_centers` 테이블(0행)을 바라보게 되어, 어린이집/초등학교 주변 규제지가 제대로 배제되지 않고 침범한 후보지를 추천하는 버그가 있었습니다.
  2. 스캔 부지 수량이 단 15개(`LIMIT 15`)로 한정되어 있어, 클릭 지점 근처의 밀집된 부지들만 스캔함으로써 주변 교통, 민원, 인구 등의 데이터 편차가 거의 나지 않아 "주변 환경이 크게 고려되지 않았다"는 부작용이 있었습니다.
- **해결책**:
  1. SQL `NOT EXISTS` 배제 필터링 대상을 실제 실물 정제데이터(649행)가 완벽히 탑재된 **`restricted_zones`** 테이블로 매핑을 수정하고, 'school'(200m), 'childcare_center'(30m), 'nosmoking_zone'(10m) 등 유형별 임계치에 맞는 공간 인덱스(GIST) 필터를 정밀하게 구축했습니다.
  2. 스캔 후보지 범주를 **`LIMIT 150`**으로 10배 확장하여 1km 인근의 넓은 지역에서 다양한 공간적 조건(유동인구 집중지, 민원 다발지 등)을 가진 부지들을 다양하게 수집한 후 AHP 가중합 연산을 거치도록 변경하여 주변 환경 분석 퀄리티를 최적화했습니다.
