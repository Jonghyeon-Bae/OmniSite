# 📋 가변적 AHP 지표 및 소수점 정밀 슬라이더 구현 계획서 (v3.0.0)

본 계획서는 지표 수량을 특정 개수(5개)로 제한하지 않고 AI 감리가 상황에 맞게 3~8개 사이로 가변 도출하며, 미세 조정 시 C.R. 불합격으로 인해 시스템이 마비되는 현상을 극복하기 위해 **"슬라이더 소수점 정밀화 및 $n$차 차수별 R.I. 난수 지수 동적 매핑 엔진"**을 설계한 3단계 고도화 계획서입니다.

---

## 1. 🔍 기술적 개편 및 해결 방안

### 1.1. 가변적 지표 수량 ($n$차) 지원 및 R.I. 동적 매핑

- **원리**:
  - AI 감리 결과에 따라 의사결정 지표 수량($n$)이 3개에서 최대 8개 이상으로 유동적으로 바뀝니다.
  - 지표 수량($n$)이 바뀜에 따라 AHP 일관성 검정 공식($C.R. = C.I. / R.I.$)에 대입할 **R.I.(Random Index) 난수 지수를 동적으로 참조**합니다.
- **R.I. 동적 매핑 테이블 (백엔드 탑재)**:
  ```python
  RI_TABLE = {
      1: 0.00, 2: 0.00, 3: 0.58, 4: 0.90, 5: 1.12,
      6: 1.24, 7: 1.32, 8: 1.41, 9: 1.45, 10: 1.49
  }
  ```

  - 분석 목적에 맞춰 감리 에이전트가 추천한 지표 수량($n$)의 크기에 부합하는 R.I. 값을 획득하여 정확한 C.R.을 계산합니다.

### 1.2. AHP 슬라이더 소수점 정밀화 (Slider Resolution & Precision)

- **원인 분석**:
  - 기존의 1~9 정수 단위 척도는 한 칸만 움직여도 비교 가중치가 2배, 3배 급격하게 요동쳐 쌍대비교 일관성이 쉽게 붕괴(C.R. >= 0.1)되고, 분석 잠금이 통제 불능에 이르는 부작용이 있었습니다.
- **소수점 단위 미세 조정 (step="0.1")**:
  - 프론트엔드 슬라이더의 `step` 속성을 `0.1`로 촘촘히 쪼개어 실무자가 미세한 가중치 차이(예: 1.5, 2.2 등)를 조절할 수 있도록 합니다.
  - 백엔드의 AHP 연산 모듈 또한 실수(Float) 비교 행렬 연산을 지원하여 일관성(C.R.)의 급격한 붕괴를 예방하고, 합리적인 조작 한계를 유연하게 도출합니다.

- **실수 행렬 기반 C.R. 연산 로직 구현**:
  - 가변 지표 수량($n$) 및 `RI_TABLE` 연동.
  - 입력받은 Float 가중치를 바탕으로 정규화 고유벡터(Eigenvector) 및 최대 고유값($\lambda_{max}$) 연산.
  - 일관성 지수 $C.I. = (\lambda_{max} - n) / (n - 1)$ 및 $C.R. = C.I. / R.I.$ 도출.
  - C.R. < 0.1 이하 조건 검증 통과 시 `ahp_models` DB 락킹.
