# 실증 준공사례 대시보드 탭 분리 리스트업 설계서 (v1.2.0-alpha-TabList)

본 설계서는 조장(USER)의 의견에 따라, RAG에 누적 적재된 외부 실증 준공사례(`verified_precedents`) 데이터를 대시보드 화면상에 일목요연하게 시각화하기 위해, **"기존 의사결정 이력 테이블 내에 가로 스위칭 탭([모의 심의 이력] / [실증 준공 사례])을 구성하여 통합 리스트업"**하기 위한 정밀 설계 문서입니다.

---

## ⚖️ 1. 시니어 개발자 관점의 UX 타당성 분석

### 1안: 하나의 테이블 공간 내 탭(Tab) 스위칭 방식 (★ 피어 적극 권장안)
- **화면 리소스 효율성**: 대시보드의 제한된 면적(Real Estate) 내에 별도의 신규 표를 아래로 덧붙이면 화면이 조잡해지고 불필요한 세로 스크롤을 강제합니다. 탭 하나로 영역을 스위칭하는 것이 프리미엄 다크모드 디자인 테마에 가장 적합합니다.
- **개념의 정합성**: 두 데이터군 모두 **"특정 필지의 입지 적합성과 사후 준공 적합성을 조회"**한다는 최종 사용자 시나리오가 동일하므로, 동일 영역 내 탭 교체가 직관성이 가장 높습니다.

---

## 🛠️ 2. 기술적 구현 설계 명세

### 1) 백엔드 실증사례 조회 API 신규 구현 (`/api/v1/spatial/precedents`)
- **기능**: `verified_precedents` 테이블을 쿼리하여 적재 완료된 성공 사례 목록을 반환합니다.
- **시맨틱 PNU/지번 파싱 피딩**: 데이터베이스 추가 스키마 파괴 없이, OCR 텍스트 내에 보관된 주소와 19자리 PNU 코드를 정규식으로 백엔드가 실시간 파싱하여 리스트 정보로 완성해 반환합니다.
  - **추출 식**: `PNU: \d{19}`, `지번 주소: [가-힣]+동\s*\d+(?:-\d+)?`

### 2) 프론트엔드 대시보드 탭 및 테이블 스위칭 구현 (`dashboard/page.js`)
- 의사결정 이력 섹션 상단에 탭 버튼 2종을 구현합니다:
  - **`[모의 심의 이력 (${historyList.length}건)]`**
  - **`[실증 준공 사례 (${precedentList.length}건)]`**
- 탭 스위칭에 따라 테이블 컬럼 헤더와 셀 렌더링 방식을 다이나믹 전환합니다.
- 실증 준공사례 행 클릭 시, 해당 준공문서의 OCR 텍스트 요약 및 시나리오 검증 보고서가 우측 RAG 패널에 즉각 연동되어 노출되는 인터랙티브 기능을 완성합니다.
